# Output Spec — XDS-001 산출물 형식 정의서

- 작성일: 2026-05-12
- 작성자: 유혜원
- 상태: Draft v0.1
- 이슈키: XDS-001
- 참조: PRD §3.G, Token Schema §8

> **TL;DR** 도구가 export하는 모든 파일의 정확한 양식과 패키지 구조. 단일 zip 패키지 안에 토큰(4종)·컴포넌트 명세 MD·UX 가이드·접근성 보고서·Tailwind preset·Figma Tokens Studio·TS 타입·Storybook 호환 데이터가 포함된다. 모든 파일은 동일한 명명 규칙(`파일종류_v버전_YYYYMMDD.확장자`)과 read-only 원칙을 따른다.

## 이 문서가 묻는 결정

- [x] **출력물의 권위**: 도구 UI를 SSoT(Single Source of Truth)로 명시, export된 파일은 read-only — 결정
- [x] **파일명 규칙**: `{파일종류}_v{SemVer}_{YYYYMMDD}.{확장자}` — 결정
- [x] **버전 충돌 처리**: 같은 날 중복 export 시 `_HHmm` 시간 suffix 자동 추가 — 결정
- [x] **패키지 형태**: 단일 zip + 그 안에 모든 파일 + manifest.json — 결정

## 1. 핵심 원칙

### 1.1 Read-Only 원칙
도구 UI가 SSoT(Single Source of Truth)이다. Export된 모든 파일은 read-only로 간주되어야 한다.

- 개발자가 export된 `tokens.css`를 직접 수정하면 다음 export에서 덮어쓰여짐
- 모든 export 파일의 첫 줄 또는 상단 주석에 다음 명시:
  ```
  /* GENERATED BY XDS Generator v{generatorVersion} — DO NOT EDIT MANUALLY */
  /* To modify, edit the project in the XDS Generator and re-export */
  ```

### 1.2 결정 가능성
도구는 동일한 입력(프로젝트 상태)에서 동일한 출력을 생성한다. 비결정적 요소(타임스탬프 외) 금지.

### 1.3 격리·완전성
하나의 zip 패키지는 외부 의존 없이 *자체로 완결*된 디자인 시스템이어야 한다. 다른 파일·서비스 참조 금지.

## 6. 컴포넌트 명세 MD 양식

각 컴포넌트는 동일한 9개 섹션을 갖는다.

```markdown
# Button

> XDS Generator v{generatorVersion} | 프로젝트: {projectName} | 버전: {semver}

## 1. 정의

사용자가 클릭으로 액션을 트리거하는 기본 컨트롤.

## 2. Anatomy (부품 구성)

| 슬롯 | 필수 | 설명 |
|---|---|---|
| `root` | ✓ | 버튼 컨테이너 |
| `label` | ✓ | 텍스트 |
| `iconLeading` | – | 좌측 아이콘 (선택) |
| `iconTrailing` | – | 우측 아이콘 (선택) |
| `spinner` | – | loading 상태일 때 표시 |

## 3. Variants

| 차원 | 옵션 |
|---|---|
| variant | `primary` \| `secondary` \| `outline` \| `ghost` \| `link` \| `danger` |
| size | `sm` \| `md` \| `lg` |
| state | `default` \| `hover` \| `active` \| `disabled` \| `loading` \| `focus` |

총 조합: 6 × 3 × 6 = 108 (단, 일부 조합은 의미상 비활성)

## 4. 토큰 매핑

### variant: primary

| 속성 | 토큰 |
|---|---|
| background | `{alias.action.primary}` |
| color | `{alias.text.onPrimary}` |
| borderRadius | `{map.radius.md}` |
| hover.background | `{map.color.primary.hover}` |
| active.background | `{map.color.primary.active}` |
| focus.ringWidth | `{alias.focus.ring.width}` |
| focus.ringColor | `{alias.focus.ring.color}` |
| disabled.opacity | `0.5` |

### size: md
| 속성 | 토큰 |
|---|---|
| height | `{alias.control.height.md}` (32px) |
| paddingX | `{alias.control.padding.x}` (12px) |
| fontSize | `{map.font.size.sm}` (14px) |

[다른 variant·size 조합 반복]

## 5. 사용 케이스 (Do)

- 주요 액션 1개에 `primary` 사용 (한 화면에 primary 1개 원칙)
- 위험한 액션(삭제·취소불가)에 `danger` 사용
- 보조 액션에 `secondary` 또는 `outline`
- 인라인 텍스트 액션에 `link`

## 6. 사용 금지 (Don't)

- 한 화면에 `primary` 버튼 여러 개
- `primary`와 `danger`를 나란히 배치 (사용자 혼동)
- 버튼 라벨 6단어 초과 (긴 문장은 다른 컴포넌트 사용)
- 아이콘만 있는 버튼은 `IconButton` 컴포넌트 사용 (Button 아님)

## 7. 접근성

- **WCAG 2.2 AA**: 포커스 표시 항상 가시 (focus.ringWidth ≥ 2px), 대비비 통과 (foreground/background 4.5:1)
- **KWCAG 2.2**: 키보드만으로 모든 액션 트리거 가능, 스크린리더 라벨 (`aria-label` 또는 가시 텍스트), 한국어 라벨 시 조사·존대 일관성
- **터치 타겟**: 모바일 사이즈는 최소 44×44px
- **prefers-reduced-motion**: hover·active transition 0ms
- **상태 전달**: `disabled`·`loading` 상태가 색뿐 아니라 아이콘·텍스트로도 전달

## 8. 코드 예시 (참고)

### React (참고용 골격, 실제 구현체는 별도)
```jsx
<Button variant="primary" size="md" onClick={handleSubmit}>
  저장
</Button>
```

### HTML + CSS Variables
```html
<button class="xds-button xds-button--primary xds-button--md">저장</button>
```

```css
.xds-button--primary {
  background: var(--xds-action-primary);
  color: var(--xds-text-on-primary);
  height: var(--xds-control-height-md);
  border-radius: var(--xds-radius-md);
}
```

## 9. 관련 컴포넌트

- IconButton — 아이콘만 있는 버튼
- ButtonGroup — 버튼 묶음
- DropdownButton — 드롭다운 동반 버튼
- Link — 텍스트 링크 (네비게이션 목적)
```

## 11. 검증 체크리스트 (Export 직전 도구가 자동 점검)

|---|---|
| 모든 컴포넌트가 Alias 토큰만 참조 (리터럴 직접 사용 없음) | 위반 시 export 차단, 위반 컴포넌트 목록 표시 |

| **Experimental 토큰이 Stable 컴포넌트에서 참조됨** | 경고 |
| WCAG 2.2 AA 모든 항목 통과 | 위반 시 export 차단 |
| KWCAG 2.2 모든 항목 통과 | 위반 시 export 차단 |
| **manifest.json의 모든 files[] 항목에 SHA-256 체크섬 생성 완료** | 누락 시 export 차단 |
| 모든 컴포넌트 명세 9섹션 채워짐 | 미채움 시 경고(차단은 아님) |

| **v0.2 이상: ux-guide.md·content-guide.md 포함 여부** | v0.2+ 프로젝트에서 누락 시 경고 |
