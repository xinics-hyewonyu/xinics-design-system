## 1. 제품 개요

### 1.1 한 줄 정의
디자이너·PD가 GUI에서 디자인 의도를 선택·조정하면, 표준 형식의 디자인 시스템 산출물(토큰·컴포넌트 명세·UX 가이드·접근성 보고서)과 접근성 자동 검증 결과를 생성·내보내는 사내 웹 도구.

### 1.2 존재 이유
PCD-001 §3 Problem이 정의한 세 가지 문제를 해소하기 위함이다.

1. 디자이너·PD가 신규 프로젝트마다 디자인 시스템을 손으로 작성하면서 1~2주를 소비하고, 그동안 개발팀의 컴포넌트 구현이 지연된다.
2. 산출물 형식이 작성자·시점마다 달라 개발자가 인수 시 매번 재해석·변환 작업을 거치며, 이 과정에서 시각 회귀 결함이 반복 발생한다.
3. 작성 과정에 접근성 검증이 명시적으로 박혀있지 않아 출시 직전 인증 심사에서 동일 유형의 미달 사항이 반복 적발되고, 교육·공공 도메인 제품의 납품 일정이 지연된다.

해결 방향은 PCD-001 §4 — 디자인 시스템 작성 모델을 *문서 직접 작성*에서 *도구 생성*으로 전환하고, 산출물 표준화·접근성 자동 검증·이력 보존을 도구가 책임진다.

### 1.3 타겟 사용자
USD §1 참조 — 직접 사용자 5종(디자인 시스템 오너·디자이너·PD·개발자·PM) + 간접 사용자 1종(최종 사용자).

### 1.4 핵심 가치 명제
1. **시간** — 신규 프로젝트 디자인 시스템 작성 시간 1~2주 → 1~2시간
2. **일관성** — 모든 산출물이 동일 형식, 동일 명명 규칙, 동일 버전 체계
3. **접근성** — WCAG 2.2 AA + KWCAG 2.2를 산출 전 단계에서 자동 강제
4. **이력** — 누가·언제·왜 결정했는지 영구 보존, 거버넌스 기반

### 3.2 기능 상세 (핵심 5개만 본 문서에서 정의, 나머지는 FRD에서 상세화)

#### C1·C2·C3 — Token Editing
- **Seed 입력**: colorPrimary·colorSuccess·colorWarning·colorError·colorBgBase·colorTextBase·borderRadius·fontSize·fontFamily 9개를 GUI로 직접 입력
- **Map 자동 파생**: Seed에서 OKLCH 색공간 기반 11단계(50~950) 자동 생성. 디자이너가 결과 확인 후 일부 단계만 수동 override 가능 (override는 잠금 표시)
- **Alias 의미 매핑**: 컴포넌트가 참조할 의미 토큰(예: `text.body`, `surface.elevated`)을 Map 토큰 중 하나에 매핑. 기본 매핑은 자동, 디자이너가 변경 가능
- **응답성**: 토큰 슬라이더 조정 후 프리뷰 갱신 16ms 이내

#### F1·F2 — Accessibility Validation
- **WCAG 2.2 AA 점검 항목**: 색상 대비비(본문 4.5:1·대형 텍스트 3:1·UI 컴포넌트 3:1), 포커스 가시성, 키보드 작동성, 폼 라벨, 헤더 위계, 링크 텍스트, 시각 정보 비-색상 표현
- **KWCAG 2.2 점검 항목**: 위 WCAG 항목 + 한국형 추가 항목(대체 텍스트, 한국어 가독성, 콘텐츠 자동 갱신 통제, 깜박임)
- **점검 시점**: (1) 토큰 변경 직후 실시간, (2) Export 직전 강제, (3) 프로젝트 열 때 캐시 결과 표시
- **결과 표시**: 미달 항목은 빨간 배지, 통과는 녹색. 각 항목 클릭 시 "어떤 토큰 때문에 왜 미달인지" 설명
- **자동 보정 제안** (v0.2): 본문 대비비가 3.8:1이면 colorTextBase를 N단계 어둡게 했을 때 4.5:1을 통과한다는 시뮬레이션 제안

#### G7·G8 — Export Package + Pre-export Review
- **전체 패키지 zip 구조**: Output Spec 문서에서 상세화
- **Pre-export Review**: 디자이너가 화면에서 본 컴포넌트는 보통 L1·L2(최대 19종)에 한정되므로, Export 직전 *미검토 컴포넌트(L3·L4)*를 순환 표시하며 "이대로 export" 또는 "추가 조정" 선택을 강제
- **파일명 규칙**: `{파일종류}_v{버전}_{YYYYMMDD}.{확장자}`, 같은 날 중복 시 `_HHmm` 시간 suffix 자동 추가 (예: `tokens_v1.0.0_20260512_1423.json`)
- **버전 자동 증가**: MAJOR(토큰 구조 변경)·MINOR(토큰 값/컴포넌트 추가)·PATCH(문서 수정) 도구가 판단하여 자동 제안, 디자이너 확인 후 확정

#### H1·H5 — Activity Log
- **기록 시점**: 프로젝트 생성·열기·토큰 변경·프리셋 변경·Export·복제·삭제 모든 액션
- **엔티티 필드**: id(자동증가 정수, 1부터), projectName, creator(v0.1은 수동 입력 / v0.3부터 SSO 자동), createdAt(ISO 8601 UTC 저장·KST 표시), action(create|update|export|duplicate|archive), styleName, outputs[]
- **outputs[] 필드**: type(tokens|component|guide|bundle), filename, size, checksum, version, **generatorVersion** (도구 자체의 버전 — 산출물 stale 판단에 사용)
- **영구 보관**: 자동 삭제 없음. 어드민에 의한 수동 삭제도 v0.3 이후 별도 권한으로만 가능
- **v0.3 SSO 도입 시 처리**: 기존 수동 입력 creator는 "unverified" 플래그 유지, 가능하면 1회 일괄 매칭 절차 제공

## 4. 비기능 요구사항

### 4.1 성능·신뢰성
- 토큰 조정 → 프리뷰 갱신: P95 16ms 이하 (60fps)
- 컴포넌트 프리뷰 전체 렌더링 (30+종): 초기 로드 P95 1.5초 이하
- Export 패키지 생성: P95 3초 이하
- 자동 저장: 5초 디바운스, 무의식적 작업 차단 없음
- **자동 저장 실패 처리**: 저장 실패 시 즉시 사용자에게 비차단 알림(토스트 또는 상단 배너) 표시, 최대 3회 자동 재시도(지수 백오프), 재시도 모두 실패 시 변경 내용을 브라우저 로컬에 임시 보관하고 "재연결 후 동기화" 안내. 사용자가 명시적으로 무시하지 않는 한 작업 손실은 발생하지 않음.

### 4.2 도구 자체의 접근성
- 도구 UI 자체도 WCAG 2.2 AA + KWCAG 2.2 준수 (도구가 강제하는 기준을 도구 본인이 어겨선 안 됨) — **v0.1부터 적용**
- 모든 컨트롤 키보드 단축키 매핑 — v0.1
- 스크린리더 라벨 100% — v0.1
- 색맹 사용자 대응 (색에만 의존한 정보 전달 금지) — v0.1 (도구 UI 자체), v0.3 (프리뷰 색맹 시뮬레이션 도구는 E6)
- 도구 UI의 `prefers-reduced-motion` / `prefers-contrast` 미디어 쿼리 대응 — v0.1
- 모바일 뷰포트 산출물 프리뷰 접근성 검증 — v0.3 (E5와 함께)
- 산출물 프리뷰의 색맹·reduced-motion 시뮬레이션 토글 — v0.3 (E6)

### 4.3 브라우저 지원
- Chrome / Edge / Firefox / Safari 최신 2버전
- IE·구형 브라우저 미지원

### 4.4 보안
- xinics.com 도메인 SSO 강제 (v0.3 이후)
- 모든 통신 HTTPS
- 사내 프로젝트 데이터 사내 망 또는 자이닉스 통제 SaaS에 저장
- 로그는 변조 방지 (append-only)
- 외부 의존성(LLM 등) 호출 시 사내 정책 검토 필요

### 4.5 확장성
- 컴포넌트 추가가 도구 코드 수정 없이 데이터로 가능한 구조
- 새 프리셋 추가가 JSON 파일 추가만으로 가능
- 새 언어 추가가 토큰 구조 변경 없이 가능 (i18n 확장 구조)
- 새 산출물 형식 추가가 export 모듈 단위로 격리

### 7.2 가정
- 자이닉스 4개 제품 라인 구성이 v1.0 기간 내 큰 변동 없음
- 한국어 본문 가독성 기준이 현행 KWCAG 2.2 수준으로 유지
- 디자이너·PD가 도구 학습에 평균 2~4시간 투자 가능

### 7.3 리스크와 완화
| 리스크 | 영향 | 완화 |
|---|---|---|
| 도구 학습 곡선이 디자이너 진입을 막음 | 도구 도입 실패 | v0.1부터 온보딩 플로우·예제 프로젝트 제공 |
