- 작성일: 2026-05-12
- 작성자: 유혜원
- 상태: Draft v0.1
- 이슈키: XDS-001

> **TL;DR** WCAG 2.2 AA와 KWCAG 2.2 두 기준을 *도구 자체*와 *도구가 생성하는 모든 산출물*에 횡단 강제하는 규칙. 64개 점검 항목, 자동 검증 + 자동 보정 제안 + Export 차단 정책. 도구가 어기지 않아야 할 기준을 도구가 생성한 디자인 시스템에도 그대로 적용.

## 1. 적용 범위 (Two Surfaces)

### 1.1 Surface A — 도구 자체 (XDS Generator UI)
도구를 사용하는 디자이너·PD·개발자·PM 등 내부 사용자에게 적용. 도구 UI가 키보드 접근·스크린리더·고대비 등 보조 기술에 대응해야 한다.

### 1.2 Surface B — 도구가 생성한 산출물 (Generated Design System)
도구가 export한 디자인 시스템이 적용된 *최종 제품*의 최종 사용자(학습자·교강사·운영자·일반 시민)에게 적용. 도구가 산출하는 토큰·컴포넌트 명세·접근성 보고서가 WCAG/KWCAG를 충족해야 한다.

### 1.3 두 Surface의 관계
**"도구가 어겨선 안 되는 기준을 도구가 생성한 결과물도 어기지 않도록"** — 도구 UI 자체가 WCAG AA 미달이면, 그 도구가 만든 결과물도 신뢰할 수 없다. Surface A와 B는 동일한 기준이 적용된다.

## 2. 기준 체계

### 2.1 WCAG 2.2 AA
W3C Web Content Accessibility Guidelines 2.2. 4개 원칙(POUR) 13개 지침 78개 성공 기준 구조 중 AA 이하 50개를 강제.

### 2.2 KWCAG 2.2
한국형 웹 콘텐츠 접근성 지침 2.2. 24개 검사 항목 + WCAG와 중복되지 않는 한국 특화 항목 (자간·행간·자동 갱신·깜박임·한국어 라벨).

### 2.3 둘의 관계
KWCAG 2.2는 WCAG 2.2를 기반으로 하되 한국 환경(한국어 가독성·정보통신접근성 인증 의무)을 반영한 추가 항목 포함. 본 도구는 *교집합이 아니라 합집합*을 강제한다.

## 4. 자동 검증 매커니즘

### 4.1 도구 자체 (Surface A) 검증

도구 빌드 파이프라인에 통합:
- **외부 엔진**: axe-core (오픈소스 a11y 엔진) — WCAG AA 자동 점검 / lighthouse CI — 성능·접근성 통합 점검
- **자체 한국어 룰 (담당 범위)**:
  - K-01·K-02 자간·행간 (CSS 토큰 자동 점검)
  - K-03 띄어쓰기·맞춤법 (외부 한국어 NLP 라이브러리 호출, v0.3+)
  - K-04 자동 갱신 통제 (컴포넌트 메타에 일시정지 지원 명시 여부)
  - K-05·K-09 한국어 존대·약어 (마이크로카피 패턴 매칭)
  - K-07 한국어 폰트 스택 (Pretendard 등 포함 여부)
  - P-08·P-09 한국어 본문 자간·행간 (K-01·K-02와 연동)
- **단위**: 페이지·컴포넌트별 점검, 모든 라우트 통과 시에만 PR 머지 가능
- **CI 차단**: WCAG AA·KWCAG 위반 발견 시 빌드 실패
- **주기**: 매 PR + 일일 정기 검사

### 4.2 산출물 (Surface B) 검증

도구가 Export 직전에 자동 수행:
- **색상 대비비**: Alias 토큰 페어 자동 점검 (`alias.text.body × alias.surface.base` 등 30+ 페어)
- **포커스 가시**: 모든 포커스 가능한 컴포넌트의 `focus.ring.width ≥ 2px` 확인
- **터치 타겟**: 모든 인터랙티브 컴포넌트의 최소 크기 확인
- **라벨**: 컴포넌트 명세서의 a11y 섹션에 `aria-label` 또는 가시 라벨 명시 확인
- **한국어 가독성**: 본문 토큰의 자간·행간 KWCAG 범위 확인
- **자동 갱신**: 명세서에 자동 갱신 컴포넌트가 있으면 일시정지 지원 명시 확인
- **이미지 대체 텍스트**: 산출물에 포함된 일러스트·아이콘에 alt 메타 확인

### 4.3 검증 결과 처리

| 결과 | 처리 |
|---|---|
| 모두 통과 | Export 버튼 활성화, "✅ WCAG·KWCAG 통과" 표시 |
| WCAG 또는 KWCAG 미달 | Export 버튼 비활성화, 미달 항목 목록 표시, 각 항목 클릭 시 어떤 토큰·컴포넌트가 원인인지 표시 |
| 경고 항목 (강제 X) | Export 가능, 경고 카운트 표시 |

## 5. 자동 보정 제안 (v0.2+)

미달 항목이 *알고리즘적으로 해소 가능*한 경우 도구가 보정값 제안.

### 5.1 색상 대비비 보정
- 본문 `alias.text.body`와 `alias.surface.base`의 대비비가 3.8:1 (미달)이면, `text.body`의 OKLCH 명도를 단계적으로 어둡게 시뮬레이션하여 4.5:1을 통과하는 가장 가까운 값 제안
- 디자이너가 [적용] 또는 [무시] 선택
- 도구가 자동으로 보정하지 않음 (디자인 의도 침해 방지)

### 5.2 포커스 링 굵기 보정
- `alias.focus.ring.width`가 1px이면 2px로 제안
- 동시에 `focus.ring.color`의 배경 대비비 점검

### 5.3 터치 타겟 보정
- 모바일 breakpoint에서 `Button.size.sm` 높이가 32px이면 44px 제안 (KWCAG 모바일 강화)
- 데스크탑에서는 32px 허용

### 5.4 자간·행간 보정 (권장 수준)
- 본문 자간이 0em이면 한국어 가독성 향상을 위해 -0.01em *권장*. 0em도 K-01 허용 범위 내이므로 의무 보정 아님.
- 행간이 1.5이면 1.6 *권장*. 1.5도 K-02 최소 기준이므로 미달 아님.
- §10 매트릭스의 "자간·행간 — 경고만" 정책과 일관됨.

### 5.5 제안 옵트인 규칙
- 모든 제안은 디자이너가 명시적으로 [적용] 버튼을 눌러야 반영
- 디자이너가 [무시]하면 *해당 미달 항목은 그대로 export 차단 상태 유지*
- 도구는 다른 방향의 해소 방법(예: 색이 아닌 폰트 굵기 조정)도 제안

## 6. 산출물에 포함되는 접근성 보고서

Export 시 자동 생성 (Output Spec §7 양식). 본 문서의 64개 항목별 통과·미달·경고 결과가 자동 채워짐.

리포트에 포함되는 메타:
- 검증 시점 (ISO timestamp + KST)
- 도구 버전 (`generatorVersion`)
- 검증 엔진 버전 (axe-core·자체 룰 버전)
- 자동 보정 적용 내역 (어떤 토큰을 어떻게 바꿨는지)
- 비검증 항목 (예: 콘텐츠 차원 검증이 필요한 항목은 "수동 검토 필요" 라벨)

---

## 7. 도구 자체의 접근성 체크리스트 (Surface A 전용 추가)

도구가 *디자이너 대신* 접근성을 강제하는 만큼, 도구 본인이 모범적이어야 한다. UI Spec §8에 정의된 항목 + 본 명세서의 §3 항목 전체.

추가 체크:
- 첫 진입 사용자에게 도구의 접근성 기능 안내 (온보딩 4단계 중 1단계)
- 키보드 단축키 시트(Cmd+/)가 항상 도움말에서 접근 가능
- 스크린리더 사용자에게 친화적인 초기 포커스 (skip-to-content 링크)
- 색상 슬라이더는 OKLCH 좌표를 텍스트 입력으로도 가능 (시각 의존 회피)
- 접근성 점검 결과는 색 + 텍스트 + 아이콘 3중 표현 (✅/⚠/❌ + 라벨)

---

## 8. 거버넌스·운영

### 8.1 KWCAG 정책 변경 대응
한국 정보통신접근성 인증 정책은 갱신됨. 도구는 점검 항목을 *데이터 파일*로 분리하여 코드 변경 없이 갱신 가능 (PRD §7.3 "KWCAG 점검 항목 변경" 리스크와 연계).

### 8.2 검증 엔진 업데이트
axe-core·lighthouse 등 외부 엔진의 버전 업데이트 시 회귀 테스트 통과 후 적용.

### 8.3 책임 분담
| 책임 | 담당 |
|---|---|
| WCAG·KWCAG 기준 해석·갱신 | 디자인 시스템 오너 (P-01) + 외부 접근성 전문가 (자문) |
| 검증 엔진 통합·CI 운영 | 개발팀 |
| 도구 자체 접근성 점검 | 도구 빌드 파이프라인 + 분기별 수동 감사 |
| 산출물 접근성 점검 | 도구 자동 검증 + 디자이너 검토 |
| 인증 심사 대응 | 디자인 시스템 오너 + 법무 |

### 8.4 인증 심사 대응
- 한국 정보통신접근성 인증 심사 신청 시 본 도구 산출물을 *심사 자료*로 활용 가능
- 도구의 접근성 보고서가 심사관의 1차 검토 자료가 됨
- 단, 콘텐츠 차원 항목(자막·약어 풀이 등)은 별도 검증 필요

---

## 9. 비스코프

- 청각·운동 보조 기술의 *완전 자동* 검증 — 기술적으로 어려운 항목(스크린리더 실제 음성 정확도 등)은 수동 검증으로 처리
- 외부 라이브러리(자이닉스 도구가 import하는 모든 라이브러리)의 접근성 검증 — 도구가 생성하는 컴포넌트 명세에만 책임
- 한국 외 다국어 접근성 (영문 WCAG는 자동 통과, 일본어 등 추가 시 별도 검증 필요)
