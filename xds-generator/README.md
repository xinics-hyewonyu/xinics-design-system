# XDS Generator

자이닉스 디자인 시스템 생성 도구(Xinics Design System Generator)의 Sprint 1A 기반 셋업.

13_Build_Kickoff.md §5의 Sprint 1A 8개 티켓(T-001~T-007) 범위만 우선 구현.

## 폴더 구조

```
xds-generator/
├── src/
│   ├── tokens/
│   │   ├── types.ts                  ← Token Schema §1~§3 (Seed·Map·Alias·W3C)
│   │   ├── build.ts                  ← Seed → Map → Alias → TokenTree
│   │   ├── algorithms/
│   │   │   ├── oklch.ts              ← OKLCH 색공간 + 11단 스케일 + 다크 파생 + 대비비
│   │   │   └── scales.ts             ← size·radius·font 스케일
│   │   └── __tests__/                ← vitest 단위·통합 테스트
│   ├── output/
│   │   ├── cssVariables.ts           ← tokens.css (CSS Variables) 생성기 — Output Spec §5.2
│   │   └── w3c.ts                    ← tokens.json (W3C 표준) 생성·역파싱 — Output Spec §5.1
│   ├── presets/
│   │   └── default.ts                ← Default 프리셋 — Style Preset Catalog §3.1
│   ├── design/
│   │   └── tokens.css                ← 도구 자체 디자인 토큰 (shadcn-like, UI Spec §9)
│   ├── version.ts
│   ├── App.tsx                       ← 통합 미리보기 UI (Seed·11단 스케일·CSS 출력)
│   └── main.tsx
├── scripts/
│   └── demo-export.ts                ← Default 프리셋 실제 파일 생성 (T-007 검증)
├── index.html
├── package.json
├── tsconfig.json
├── tsconfig.node.json
├── vite.config.ts
├── tailwind.config.js
└── postcss.config.js
```

## 실행

```bash
npm install
npm run dev            # 개발 서버 (브라우저로 통합 미리보기 확인)

npm run test           # vitest 단위·통합 테스트
npm run demo:export    # ./dist-sample/ 에 Default 프리셋 export
```

## 핵심 설계 결정

- **OKLCH 1순위**: 11단 스케일·다크 파생·대비비 계산이 모두 인지적으로 균일한 OKLCH 위에서 수행됨 (Token Schema §1·§4.1).
- **Alias = Map 참조**: Alias 토큰은 `{map.color.primary.default}` 같은 reference 문자열만 허용. 리터럴 직접 보유 금지 (Token Schema §2.3).
- **Read-only 출력**: 모든 export 파일 상단에 `GENERATED BY ... DO NOT EDIT MANUALLY` 헤더 (Output Spec §1.1).
- **도구 토큰 분리**: 도구 자체의 UI 토큰(`src/design/tokens.css`)은 도구가 생성하는 시스템과 완전히 분리됨 (UI Spec §9).
