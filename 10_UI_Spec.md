# UI Spec — XDS-001 도구 화면 명세

- 작성일: 2026-05-12
- 작성자: 유혜원
- 상태: Draft v0.1
- 이슈키: XDS-001
- 참조: PRD §3 전체, Token Schema, Output Spec, Style Preset Catalog

> **TL;DR** 디자인 시스템 생성 도구 자체의 화면 명세. 첨부 이미지 두 장(좌측 스타일 리스트 + 중앙 컴포넌트 프리뷰 / 우측 Customize Theme 슬라이더)을 결합한 단일 에디터 화면이 핵심. 7개 주요 화면(로그인·대시보드·에디터·Export·접근성 리포트·AI Generate·설정) + 도구 자체의 WCAG 2.2 AA 준수.

## 이 문서가 묻는 결정

- [x] **메인 레이아웃**: 4-panel 에디터(좌 스타일·중앙 프리뷰·우상 토큰·우하 접근성) — 결정
- [x] **데스크탑 우선**: 최소 너비 1280px, 1440px 기준 — 결정 (디자이너 작업 환경)
- [x] **다크/라이트**: 도구 자체도 다크모드 지원, 자동 감지 + 수동 토글 — 결정
- [ ] **반응형 하한**: 태블릿(768px) 지원 / 미지원 — *공개 질문*

### 4.2 좌측 패널 — 스타일 프리셋 리스트

- Style Preset Catalog의 7개 프리셋 + AI Generate를 단일 flat 리스트로 표시
- 각 프리셋은 시각 톤 기반 한글 라벨 (예: "둥근·친근", "미니멀", "트렌디", "각진·정밀")
- 역할명(학습자·교수자·운영자 등) 사용 금지 — 회사 제품이 보통 이 역할들을 모두 다루므로 변별력이 없음
- **프로젝트 타입별 추천 ★ 표시**: 현재 프로젝트의 타입(`admin`/`lms`/`ai`/`b2c`)에 따라 Style Preset Catalog §5의 1순위 프리셋이 ★ 아이콘으로 강조 (선택 강제 X)
- 추천 프리셋이 리스트 상단으로 자동 정렬
- 단일 선택, 클릭 시 즉시 중앙·우측 패널 갱신
- 첨부 이미지 1과 유사한 느낌 (선택 시 파란 테두리)
- 각 항목 hover 시 5초 미니 미리보기 (300ms delay)

### 4.3 중앙 패널 — 컴포넌트 프리뷰

- **상단 토글**: L1 핵심(5종) / L2 확장(14종) / L3 전체(30+종)
- **컴포넌트 표시**: 첨부 이미지 1의 우측 영역처럼 컴포넌트 그리드
- **상호작용**: 컴포넌트 클릭 시 우상 패널이 *해당 컴포넌트의 토큰 매핑*으로 포커스 (예: Button 클릭 → "Button color = primary")
- **컴포넌트별 상태 표시**: hover·active·disabled 상태도 동시에 한 화면에 (Storybook 매트릭스 스타일)
- **모바일 프리뷰 토글** (v0.3+ E5): 우상단 [🖥 / 📱] 토글
- **시뮬레이션 토글** (v0.3+ E6): [🎨 색맹 ▼] [💨 reduced motion]

### 4.4 우상 패널 — Customize Theme 슬라이더

첨부 이미지 2의 우측을 차용. **Quick Access (항상 노출) + 고급 설정 (펼침)** 2단 구성.

**Quick Access (디자이너가 가장 자주 만지는 4종)**:
- Primary Color: 컬러피커 (OKLCH 슬라이더 + HEX 입력 + 스와치 7색)
- Border Radius: 슬라이더 0~24px + 숫자 입력
- Density: 라디오 (Compact·Default·Comfortable)
- Font Family: 셀렉트 (Pretendard·Inter·Roboto·Geist·커스텀)

**고급 설정 (`▼ 펼치기`로 접근, Seed 9종 전체 + 보조 토큰)**:
- color.success / color.warning / color.error / color.info (시맨틱 색 4종)
- color.bgBase / color.textBase (배경·본문 색)
- fontSize (본문 기본)
- spacingBase (간격 단위)
- wireframe 모드 토글
- 다크모드 수동 override 영역 (Token Schema §4.1)

**조작**: Undo/Redo 버튼 (C6 v0.1), 변경 사항은 5초 디바운스 후 자동 저장.

**갱신 규칙**: 슬라이더 조정 → 중앙 프리뷰 16ms 내 갱신 (PRD §4.1).

### 4.5 우하 패널 — 접근성 점검

- WCAG 2.2 AA·KWCAG 2.2 항목별 통과/미달 카운트
- 미달 항목 클릭 시 *왜 미달인지* 모달
- 자동 보정 제안 (v0.2+): "본문 대비비를 4.5:1로 만들려면 textBase를 oklch(0.20→0.15)로" 버튼 한 번 클릭
- 상세 보기 → 접근성 리포트 화면 전체로 전환

## 8. 도구 자체의 접근성

도구 UI 자체도 WCAG 2.2 AA + KWCAG 2.2 준수 (PRD §4.2).

**WCAG 2.2 AA 항목**

| 요건 | 적용 |
|---|---|
| 모든 컨트롤 키보드 작동 | 사이드바·슬라이더·컬러피커·컴포넌트 클릭 모두 키보드 가능 |
| 포커스 가시 (2px 이상) | 도구 자체 토큰으로 정의 |
| 색상 대비비 | 본문 4.5:1, UI 컴포넌트 3:1 |
| 스크린리더 라벨 | 모든 아이콘 버튼에 aria-label, 슬라이더에 현재 값 announce |
| reduced-motion 대응 | OS 설정 존중 — 도구 UI·프리뷰 모두 0ms 처리 (§12와 일관) |
| 색맹 사용자 | 슬라이더·차트는 색 + 텍스트/패턴 병행 |

**KWCAG 2.2 추가 항목**

| 요건 | 적용 |
|---|---|
| 한국어 본문 가독성 (KWCAG 5.3) | 본문 자간 -0.01em, 행간 1.6 |
| 대체 텍스트 (KWCAG 1.1) | 모든 일러스트·아이콘·차트에 alt 또는 figcaption |
| 콘텐츠 자동 갱신 통제 (KWCAG 5.4) | Toast 자동 닫힘 5초 + hover 시 일시정지, 자동 캐러셀 미사용 |
| 깜박임 (KWCAG 2.3) | 초당 3회 이하 깜박임만 허용 |
| 명확한 라벨 (KWCAG 4.1) | 입력 필드는 항상 라벨·placeholder 분리 (placeholder만 사용 금지) |

## 9. 디자인 토큰 (도구 자체용 — XDS 시스템과 분리)

도구 자체의 디자인은 **XDS-Tool 자체 토큰**을 사용 (도구가 생성하는 디자인 시스템과 별개):
- 기본 톤: `shadcn-like` 차용
- 폰트: Pretendard + Inter
- 다크/라이트 모두 지원
- 토큰 파일: `xds-tool-tokens.css` (도구 코드에 내장)

이렇게 분리하는 이유: 도구가 만든 디자인 시스템을 디자이너가 미리보기 할 때, 도구 UI 자체 톤과 섞이면 안 됨.

## 10. 상태별 화면

### 10.1 로딩
- 첫 진입: 풀스크린 스피너 + "XDS Generator를 불러오는 중…"
- 프로젝트 로딩: 에디터 영역 스켈레톤
- Export 처리 중: 모달 내 진행률 표시

### 10.2 오류
- 자동 저장 실패: 상단 배너 "오프라인 모드 — 변경 사항은 로컬에 저장됩니다 [재시도]"
- LLM 호출 실패 (AI Generate): "잠시 후 다시 시도해주세요. 사용량 한도 도달이면 어드민에게 문의."
- 권한 오류 (v0.3+): "이 프로젝트에 대한 권한이 없습니다."

### 10.3 빈 상태
- 첫 사용자: 대시보드에 "첫 프로젝트를 만들어 보세요" + 4개 타입 카드
- 검색 결과 없음: "검색 결과가 없습니다. 다른 키워드로 시도해 보세요."
- 활동 로그 비어있음: "아직 작업 기록이 없습니다."

## 11. 반응형

### 11.1 데스크탑 (1280px+)
정상 4-panel 레이아웃.

### 11.2 데스크탑 좁음 (1024~1279px)
- 좌측 사이드바 자동 collapse (아이콘만)
- 우상·우하 패널이 탭으로 변경 (좁아도 폭 보존)

### 11.3 태블릿 (768~1023px)
- 권장하지 않음. 진입 시 "데스크탑 환경에서 사용을 권장합니다." 경고.
- 강행 시: 4-panel을 *탭으로 분리*하여 한 번에 하나만 표시.

### 11.4 모바일 (~767px)
- 미지원. "데스크탑에서 접속해주세요." 안내 화면.

## 12. 모션·트랜지션

| 요소 | 시간·이징 |
|---|---|
| 컴포넌트 프리뷰 갱신 | 100ms ease-out (토큰 변경 반응) |
| 모달 열기·닫기 | 200ms ease-in-out |
| 사이드바 토글 | 150ms ease-out |
| 알림 토스트 | 200ms slide-in, 자동 닫힘 5초 (hover 시 일시정지) |
| Undo/Redo 시각 피드백 | 200ms 깜박임 효과 |

`prefers-reduced-motion: reduce`일 때 도구 UI·프리뷰 모두 0ms 처리 (§8 정합).

## 13. 사용자 온보딩 (PRD §7.3 리스크 완화)

### 13.1 첫 진입
- 대시보드에서 자동 시작되는 4단계 투어
- Step 1: "여기서 프로젝트를 만드세요"
- Step 2: "에디터에서 스타일과 토큰을 조정해요"
- Step 3: "접근성이 자동 검증됩니다"
- Step 4: "완료되면 Export 하세요"
- "다시 보기" 옵션 + 설정에서 재시작 가능

### 13.2 예제 프로젝트
- 첫 사용자에게 *읽기 전용 예제 프로젝트 3개* 자동 추가
- "AICMS Admin 예제", "LMS 예제", "AI Studio 예제"
- 복제 → 자기 프로젝트로 시작 가능

### 13.3 인라인 도움말
- 모든 토큰·컴포넌트에 hover 시 1줄 설명
- "?" 버튼 클릭 시 해당 영역 상세 가이드
- `Cmd+/`로 단축키 시트 토글
